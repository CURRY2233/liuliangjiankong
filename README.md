# 🚀 云服务器流量防扣费监控 (v2.1) - 操作手册

本脚本旨在防止云服务器（如谷歌云 GCP、腾讯云、阿里云等）因为异常流量跑路或被恶意攻击导致的天价账单。当流量到达设定阈值（分为预警线和关机线）时，脚本会自动通过 **Server酱** 或 **Telegram Bot** 发送通知，若突破关机线则自动强制断电（关机），确保你的钱包安全。

---

## 🌟 一、核心特性

- **多云支持 + 时区对齐**：支持自动识别计费周期（谷歌云月初按美国太平洋时间，国内云按北京时间），确保统计无缝衔接。
- **防止误判关机**：关机阈值采取**防误判拦截**（第一次超标发高危通知并观察 5 分钟，若 5 分钟后依然超标才无情切断电源）。
- **多网卡流量聚合**：自动识别并聚合系统内所有有效网卡（支持自定义筛选）。
- **消息精简抗截断**：针对微信和 Server 酱进行通知排版优化，即使在锁屏通知栏也能完全展示所有的关键流量数字。
- **无感静默升级**：覆盖安装自动沿用旧配置及清理老残留，且保留计费数据。

## 🛠️ 二、快速安装部署

### 首要准备
建议使用 `root` 账户进行操作。跑监控需要最高权限，普通账户请务必先执行：
```bash
sudo -i
```

### 方式一：一键自动部署（推荐）
不用下载或找代码，直接在服务器终端粘贴以下命令并回车即可：
```bash
wget -qO traffic_monitor.sh https://raw.githubusercontent.com/CURRY2233/liuliangjiankong/main/traffic_monitor.sh && bash traffic_monitor.sh
```

### 方式二：纯手工直接复制粘贴
如果你喜欢掌控一切，可直接打开服务器黑框，将本脚本的**所有代码文本直接右键粘贴**进去，最后敲击一下**回车键**即可！
*(新代码内部使用了严密的 `( ... )` 圆括号进程隔离设计，完全支持无脑粗暴粘贴，绝不会出现吞字、错行或输入冲突等情况！)*

### 交互配置指引
运行后脚本会引导你进行配置，**直接回车**代表使用括号里的 `[当前: 默认值]`：

1. **选择网卡**：默认为 `1`（全部汇总），最安全全面。
2. **服务器标识**：脚本提供了诸如 `[1]谷歌云`、`[2]腾讯云` 的数字快捷菜单，直接按一下数字回车即可！完美避开 SSH 无法打出中文的问题。你想自定义的话，输入 `0` 后也支持粘贴你起好的中文名字。
3. **计费时区**：谷歌云选 `1`，腾讯/阿里选 `2`（根据服务器名会自动推荐预测）。
4. **推送鉴权**：
   - `Server酱 SendKey`: 前往 Server酱 获取，用于微信推送（不填直接回车跳过）。
   - `TG Bot Token` & `TG Chat ID`: 填入你的机器人 Token 和个人 ID，用于 TG 推送（不填直接回车跳过）。
5. **阈值设定**：
   - `预警阈值 GB`: 正常情况建议设为总限额的 80%~90%。例如限额 200G，可设为 150G。
   - `关机阈值 GB`: 服务器断电生死线。建议比预警线高一些，比如限额 200G，可设为 180G（预留一些关机执行时可能滑行的流量）。
   - `汇报间隔`: 日常平安汇报的频率，建议设 1，或 7（每周报一次）。
6. **流量方向**：
   - 谷歌云仅计算出站，选 `1`。
   - 国内大部分服务器（按流量计费计双向或者取大值），选出站 `1` 或双向 `3`。
7. **最终确认**：输入 `回车` 确认部署。

## 📱 三、消息推送场景一览

| 场景 | 触发时机 | 微信通知栏预览效果（示例） | 备注 |
| :--- | :--- | :--- | :--- |
| **测试/部署** | 安装完成时立即触发 | `谷歌云 ✅已用6G/余174G` | 证明通道畅通，且正文会带你的当前外网 IP 供你记录。 |
| **日常平安汇报** | 按你设定的汇报间隔天数 | `谷歌云 📊已用85G/余94G` | 防止你以为脚本死掉了，给你吃定心丸。 |
| **重启归来** | 服务器意外重启或正常开机时 | `谷歌云 🚀开机 已用6G` | 提示服务器已被唤醒，正文中附带 IP。 |
| **流量预警** | 当流量跨过设定的【预警线】时 | `谷歌云 ⚠️预警 已用150...` | 只发一次。此时你应该上线看看是不是受到攻击或者是正常使用到了瓶颈。 |
| **高危观察** | 当流量跨过设定的【关机线】首轮 | `谷歌云 ⚠️已达180G 观察中` | 这是最后一次警告！脚本赋予 5 分钟缓期，让你有机会去查看情况，如果下次检查仍在超限范围，则拔电源！ |
| **最终斩杀** | 五分钟高危观察期结束仍未解除 | `谷歌云 ☠️已达180G 断电!` | 通知发出的同时，执行强制安全关机 `shutdown -h now`！ |

## 🔍 四、进阶操作与内部原理

**部署后的文件路径记录图**：
- **实时运行脚本**: `/root/traffic_monitor_pro.sh`（核心监控逻辑在这里！）
- **定时任务**: `/etc/cron.d/traffic-monitor`（每 5 分钟触发一次，每月重置，开机自启全由他管）
- **日志路径**: `/var/log/traffic-monitor.log`（保留最近4周记录，每周轮转防占空间）
- **状态缓存文件**: `/root/.[各类缓存文件]` （比如记录上一次通知日期、是否已发预警等，防止重复刷屏）
- **核心依赖网络工具**: `vnstat` （用于精准抓取网卡系统底层流量，独立使用不受系统自身时间影响）

**如果你想直接验证推送**：
在服务器中敲入这条指令，测试一下通道是否正常：
```bash
/root/traffic_monitor_pro.sh test
```

## 🗑️ 五、一键卸载

如果不再需要该脚本保护，随时输入以下命令进行无残留式自动卸载：
```bash
/root/traffic_monitor_pro.sh uninstall
```
只需敲入 `yes` 进行确认，脚本会优雅地自行销毁相关的 cron 任务、日志和自身文件，挥一挥衣袖不带走一片云彩。
